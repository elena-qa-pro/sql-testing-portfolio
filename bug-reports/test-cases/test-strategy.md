# Test Strategy

## Objective
Ensure system quality, data integrity, and functional correctness across application layers.

---

## Scope
Testing includes:

- Functional testing
- API testing
- Database validation
- Negative testing
- Edge cases
- Regression testing

---

## Test Levels
- Unit Testing — covered by developers
- Integration Testing — API + DB validation
- System Testing — full workflow validation
- Acceptance Testing — business validation

---

## Test Types
- Smoke testing
- Sanity testing
- Regression testing
- Exploratory testing
- Negative testing

---

## Tools
- Postman
- SQL queries
- Browser DevTools
- Jira
- TestRail

---

## Risks
- Invalid test data
- Environment instability
- API dependency failures

---

## Exit Criteria
Testing is considered complete when:

- All critical defects fixed
- Regression passed
- No blocker defects remain
